On Mapping an Election
There are many ways to map an election, and each form represents a different kind of truth. Here, we will examine two.

The default view shows your typical choropleth map. It is colored by winner (Democrat or Republican) and darkened by how large the win margin is. So if there was a land-slide victory for Donald Trump, you would expect to see a dark-red county. Visualizing it this way (instead of a simple red for a Republican win) shows a bit more nuance. We can see how close the split is in a lot of counties. However, while it is better than a simple win/loss visualization, it still has some faults. Even by accounting for near ties, one would think that Donald Trump should have won (I see a lot of red). That's because this map fails to visualize something vitally important: population.

Since land doesn't vote, and people do*, the second map still shows each winner by color, but each county is now represented as a circle. The size of the circle is determined by the total amount of votes cast in each county. While this map does suffer from counties obscuring one another, it gives us a sense of how many votes were cast in either direction. Perhaps a better form of this map would include a graduated symbol instead, where populations are classed into certain sizes (so we can see the incredibly tiny circles in, say, the Great Plains).

*Okay fine, electors vote.
